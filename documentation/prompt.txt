## Step 2 — Rules (only after plan is approved)

Follow these rules:

### Mobile-first rules
- Write base styles for 375px — no default desktop assumptions
- Add `md:` or `lg:` overrides for desktop expansion
- Test mental model at both 375px and 1280px before marking a section done
- Touch targets: all interactive elements (buttons, pills, nav links) must be minimum 44×44px on mobile
- Font sizes use existing fluid `clamp()` tokens from `globals.css @theme {}` — do not hardcode `px` font sizes
- Never use `hover:` styles without a corresponding `active:` state for mobile tap feedback

### Structure rules (non-negotiable)
- Never add `"use client"` to `page.tsx` or layout-level Server Components
- Never hardcode content — all text/data stays in `eventConfig.ts`
- Never create new color or font tokens — use only what is defined in `globals.css @theme {}`

- Inline `style={{}}` only for dynamic values (hex from config, per-item positional data) — must include a comment explaining why
